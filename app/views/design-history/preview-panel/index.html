{% extends "design-history/template.html" %} {# --- Feature metadata --- #} {% set featureName = "Preview panel" %} {% set latestVersionLink = "/titan-mvp-1.2/form-editor/question-type/radios-nf/edit" %} {% set status = "Tested" %} {% set statusColor = "green" %} {% set userNeed = "Users need to see how their form will appear to the public so they can check content and layout in real time while editing." %} {% set phase = "Private beta" %} {% set category = "Form creation" %} {% set description %} The preview panel allows users to see what their form will look like to members of the public while building it. It updates in real time and supports list-based question types such as radio buttons, checkboxes, and repeatable question sets. You can view the design system preview panel to see the current implementation.
Key Features and Accessibility
  • Real-time updates: The preview panel updates automatically as users edit questions, labels, and guidance text.
  • Visual validation: Users can see exactly how their form will appear to members of the public, including layout, spacing, and formatting.
  • Supports all question types: Preview accurately represents radio buttons, checkboxes, text inputs, repeatable question sets, and more.
  • Accessible by default: The preview uses GOV.UK Design System components, ensuring accessibility standards are maintained.

Recent improvements:

  • Live preview panel: Shows question formatting in real time as users type.
  • Repeatable questions support: Preview accurately displays how repeater question sets will appear to users.
  • Improved clarity: Better visual distinction between edit and preview areas.
  • Enhanced guidance: Updated guidance for using repeaters and list options in the preview context.
Key features and benefits
  • Real-time feedback: See changes instantly as you edit questions, labels, and guidance.
  • Visual validation: Check layout, spacing, and formatting before publishing your form.
  • Context awareness: Preview shows how questions appear in the full form context.
  • Accessibility preview: Verify that your form meets accessibility standards visually.
  • User experience testing: Validate how the form will look and feel to end users.
  • Error prevention: Catch layout and formatting issues before going live.
What users experience
  1. Edit your form: Add questions, labels, guidance text, and configure question types.
  2. View preview: The preview panel automatically updates to show how your form looks.
  3. Check layout: Verify spacing, formatting, and visual hierarchy in the preview.
  4. Validate content: Ensure questions and guidance text appear correctly to users.
  5. Test interactions: See how repeatable questions, conditionals, and other features render.
  6. Iterate: Make adjustments based on what you see in the preview.
